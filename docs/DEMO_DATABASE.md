# Demo Database System

The Inator Platform provides two approaches for working with demo data, depending on your needs.

## 🔄 Non-Destructive Approach (Recommended for Testing)

**Use Case**: You want to test with demo data but keep your real data safe.

### Commands

```bash
# 1. Build demo databases (one time, or when you want to refresh)
task setup:demodb

# 2. Activate demo data (backs up your current databases)
task demodb:activate
task restart:all

# 3. When done, restore your original data
task demodb:deactivate
task restart:all
```

### How It Works
- Creates separate demo database files (`db-demo.sqlite3`)
- When activated, backs up your current databases to `.bak` files
- Swaps in the demo databases
- Can easily swap back to your original data

### Demo Data Included
- **4 Companies**: Acme Corporation, Globex Industries, Initech LLC, Wayne Enterprises
- **12 Users**: 2 platform admins, 10 company users (managers and members)
- **6 Catalog Items**: Security cameras, sensors, locks, alarms, NVR
- **Full RBAC Hierarchy**: ADMIN (100), MANAGER (30), MEMBER (10)

---

## 💥 Destructive Approach (Fresh Start)

**Use Case**: You want to completely wipe everything and start fresh with demo data.

### Commands

```bash
# Complete reset (asks for confirmation)
task demo:full

# Or step by step:
task demo:reset  # Purges all databases
task demo:seed   # Populates with demo data
task restart:all
```

### ⚠️ Warning
This approach **permanently deletes** your current data. It cannot be recovered.

The system will ask for confirmation before proceeding:
```
⚠️  WARNING: This will DELETE ALL DATA in all databases!
   This is a destructive operation. Your current data will be lost.

   For non-destructive demo: use 'task setup:demodb' instead.

Are you sure you want to continue? [y/N]
```

## 🛠️ Technical Details

### Database Locations

**Active Databases:**
- `Authinator/backend/db.sqlite3`
- `Userinator/backend/db.sqlite3`
- `FULFILinator/backend/db.sqlite3`
- `RMAinator/backend/db.sqlite3`

**Demo Database Copies (Non-Destructive):**
- `Authinator/backend/db-demo.sqlite3`
- `Userinator/backend/db-demo.sqlite3`
- `FULFILinator/backend/db-demo.sqlite3`
- `RMAinator/backend/db-demo.sqlite3`

**Backup Files (When Demo Active):**
- `Authinator/backend/db.sqlite3.bak`
- `Userinator/backend/db.sqlite3.bak`
- `FULFILinator/backend/db.sqlite3.bak`
- `RMAinator/backend/db.sqlite3.bak`

### Seed Scripts

The demo data is generated by management commands:
- `Authinator/backend/auth_core/management/commands/seed_demo.py`
- `Userinator/backend/users/management/commands/seed_demo.py`
- `FULFILinator/backend/*/management/commands/seed_demo.py` (minimal - items only)
- `RMAinator/backend/*/management/commands/seed_demo.py` (not yet implemented)

To run individual seed commands:
```bash
cd Authinator/backend && python manage.py seed_demo
cd Userinator/backend && python manage.py seed_demo
cd FULFILinator/backend && python manage.py seed_demo
cd RMAinator/backend && python manage.py seed_demo
```

## 📊 Quick Reference

| Scenario | Command | Destructive? | Backs Up Real Data? |
|----------|---------|--------------|---------------------|
| Test demo safely | `task setup:demodb` → `task demodb:activate` | No | Yes |
| Restore real data | `task demodb:deactivate` | No | N/A |
| Fresh start | `task demo:full` | Yes | No |
| Just seed data | `task demo:seed` | No* | No |

*`demo:seed` is not destructive if run after `demo:reset`, but assumes databases are already empty.

## 🆘 Troubleshooting

### "Backup already exists - already activated?"
You already have demo data active. Run `task demodb:deactivate` first.

### "No demo DB found - run 'task setup:demodb' first"
You need to build the demo databases before activating them. Run `task setup:demodb`.

### Services not reflecting changes
After activating/deactivating demo data, always restart services:
```bash
task restart:all
```

### Want to rebuild demo databases
If the demo data is outdated or you've made changes:
```bash
# If demo is active, deactivate first
task demodb:deactivate

# Rebuild demo databases
task setup:demodb

# Reactivate
task demodb:activate
task restart:all
```
