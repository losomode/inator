# Demo Database System

The Inator Platform provides a safe, non-destructive way to test with comprehensive demo data.

## ЁЯФД Using Demo Data

### Commands

```bash
# 1. Build demo databases (one time, or when you want to refresh)
task setup:demodb

# 2. Activate demo data (backs up your current databases)
task demodb:activate

# 3. Set up OAuth providers (if you have credentials in .env)
task setup:sso

# 4. Restart services to pick up demo data
task restart:all

# 5. When done, restore your original data
task demodb:deactivate
task restart:all
```

### How It Works
- Creates separate demo database files (`db-demo.sqlite3`)
- When activated, backs up your current databases to `.bak` files
- Swaps in the demo databases
- Can easily swap back to your original data

### What's Included
- **4 Companies**: Acme Corporation, Globex Industries, Initech LLC, Wayne Enterprises
- **12 Users**: 2 platform admins, 10 company users (managers and members)
- **6 Catalog Items**: Security cameras, sensors, locks, alarms, NVR
- **Full RBAC Hierarchy**: ADMIN (100), MANAGER (30), MEMBER (10)

### How It's Safe
тЬЕ Your real data is backed up to `.bak` files before swapping
тЬЕ Demo data is stored in separate `db-demo.sqlite3` files
тЬЕ Easy to switch back and forth between real and demo data
тЬЕ No risk of data loss

These are automatically called by `task setup:demodb`.

## ЁЯЖШ Troubleshooting

### "Backup already exists - already activated?"
You already have demo data active. Run `task demodb:deactivate` first.

### "No demo DB found - run 'task setup:demodb' first"
You need to build the demo databases before activating them. Run `task setup:demodb`.

### Services not reflecting changes
After activating/deactivating demo data, always restart services:
```bash
task restart:all
```

### Want to rebuild demo databases
If the demo data is outdated or you've made changes:
```bash
# If demo is active, deactivate first
task demodb:deactivate

# Rebuild demo databases
task setup:demodb

# Reactivate
task demodb:activate
task restart:all
```
